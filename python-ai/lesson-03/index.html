WHY FUNCTIONS

函数解决三个真实问题

减少重复

同一段逻辑不用复制很多遍。

命名逻辑

函数名告诉别人这段代码想做什么。

方便测试

给不同输入，观察输出是否符合预期。

关键句：函数是“给一组步骤起名字”。
可以展示重复 print 或重复计算，再问：哪一块值得封装？
FUNCTION SYNTAX

函数定义后不会自动运行

定义

def greet(name):
    print(f"Hello, {name}!")

调用

greet("Sponge Bob")
常见错误：只写了函数定义，但忘记在下面调用。
分清 define 和 call。定义像写菜谱，调用才是真的做菜。

CALLING FUNCTIONS

Quick check · tap an answer

如果只写 def greet(name): 但没有写 greet("Mia")，会发生什么？

函数定义只是把步骤保存起来；调用函数时，函数体才会执行。

这道题帮助学生区分函数定义和函数调用。
PARAMETERS VS ARGUMENTS

parameters 是占位名，arguments 是具体值

定义时

def add_numbers(num_1, num_2):

num_1 和 num_2 是 parameters。

调用时

add_numbers(3, 5)

3 和 5 是 arguments。

课堂问法：这个名字是在函数定义里出现，还是调用时传进去？
中文里都叫参数容易混淆，建议用“占位名/具体值”解释。

return 不等于 print：print 是显示；return 是把值传回去。
这是本课最容易混淆的地方。反复用“屏幕输出”和“交回结果”区分。

RETURN

Quick check · tap an answer

下面哪句话最准确？

return 的作用是把值交回调用处；print 的作用是显示文本。return 后同级代码不会继续执行。

如果时间允许，现场在 return 后加一行 print，验证它不会执行。
KEYWORD ARGUMENTS

关键字参数让调用更清楚，也更不怕顺序错

positional

calculate_price("apple", 3, 0.5)

短，但读者需要记住顺序。

keyword

calculate_price(item="apple", quantity=3, price=0.5)

长一点，但含义更清楚。

用 calculate_price 演示：顺序传错时可能不报错，但结果错，这比报错更危险。

SCOPE + DOCSTRING

函数内部变量通常只在函数内部有效

scope

  • 函数内创建的变量是 local
  • 外部直接访问通常会 NameError
  • 返回值是更清楚的数据出口

docstring

  • 写在函数开头的三引号字符串
  • 说明函数做什么
  • 说明输入和返回值
好函数：名字清楚，输入清楚，返回清楚，说明清楚。
scope 不必讲太深，重点是为什么函数内部变量不能随便在外面用。
